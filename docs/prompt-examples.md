# Prompt examples for Gemini 3 Series

These prompts are starting points for testing Gemini 3 Series through APIDot. Adjust brand names, subject details, source URLs, and safety rules before production use.

## 1. gemini-3-flash-preview

Write a concise launch checklist for a developer API that supports native Gemini requests.

## 2. gemini-3-pro-preview

Review this API integration plan and identify the highest-risk missing checks before production launch.

## Production prompt notes

- Store prompt versions together with task identifiers, selected model, owner, and request payload.
- Keep source media URLs reachable long enough for APIDot to fetch them.
- Be explicit about subject, composition, output constraints, quality tier, and source media.
- Do not log prompts or source media URLs if they may contain private customer data.
